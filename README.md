# QLinkr友链管理系统

QLinkr是一个轻量级、美观易用的友链管理系统，支持友链添加、审核、分类管理及个性化设置，适用于个人博客、网站的友情链接管理。


## 系统特点

- 简洁直观的管理界面，支持移动端适配
- 友链申请与审核流程自动化
- 支持友链分类、排序及状态管理
- 个性化主题设置（颜色、背景、玻璃效果等）
- 夜间模式自动切换，提升浏览体验
- 响应式设计，适配各种设备屏幕
- 内置安装向导，简化部署流程


## 技术栈

- 后端：PHP 7.4+
- 数据库：MySQL 5.6+
- 前端：HTML5、Tailwind CSS v3、JavaScript、Font Awesome


## 环境要求

- Web服务器：Apache/Nginx
- PHP版本：7.4 及以上
- MySQL版本：5.6 及以上
- 必须启用的PHP扩展：
  - PDO（数据库连接）
  - **fileinfo**（文件类型检测，核心扩展）
  - json（数据处理）
  - session（用户会话管理）


## 部署步骤

### 1. 安装必要的PHP扩展（必做）

fileinfo扩展是系统核心依赖（用于处理图片上传），必须先安装：

#### 宝塔面板用户
1. 登录宝塔面板，进入「软件商店」
2. 找到已安装的PHP版本，点击「设置」
3. 进入「扩展」选项卡，找到「fileinfo」并勾选安装
4. 安装完成后重启PHP服务

#### Linux服务器（手动安装）
- **Debian/Ubuntu**：
  ```bash
  sudo apt-get install php7.4-fileinfo  # 替换7.4为你的PHP版本
  sudo systemctl restart apache2  # 或 nginx
  ```

- **CentOS**：
  ```bash
  sudo yum install php74-php-fileinfo  # 替换74为你的PHP版本
  sudo systemctl restart httpd  # 或 nginx
  ```

#### Windows服务器
1. 找到PHP安装目录下的`php.ini`文件
2. 打开文件，搜索并取消注释（删除前面的分号）：
   ```ini
   extension=fileinfo
   ```
3. 重启Apache/IIS服务


### 2. 创建站点与数据库

1. **创建网站目录**  
   在服务器上创建站点根目录（如`/www/wwwroot/yourdomain.com`），并设置目录权限为755。

2. **创建MySQL数据库**  
   - 通过数据库管理工具（如phpMyAdmin）或命令行创建数据库：
     ```sql
     CREATE DATABASE your_dbname CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
     ```
   - 记录数据库信息（数据库名、用户名、密码），后续安装需要使用。


### 3. 上传系统文件

1. 将系统所有文件上传至步骤2创建的网站目录
2. 确保以下目录具有可写权限（用于图片上传和配置）：
   - `uploads/`
   - 网站根目录（临时用于生成配置文件）


### 4. 运行安装向导

1. 在浏览器中访问你的域名（如`http://yourdomain.com`）
2. 自动进入安装引导界面，按提示完成以下步骤：
   - **环境检测**：系统会自动检测PHP版本及扩展是否符合要求
   - **数据库配置**：填写步骤2中创建的数据库信息（主要填写数据库名、用户名、密码）
   - **完成安装**：点击「安装」按钮，系统会自动创建数据表并生成配置文件

### 5. 安装完成后的操作

1. 安装成功后，系统会自动跳转到登录页面
2. 默认账号admin，密码admin888
3. 使用默认的管理员账号登录后台（默认后台路径：`http://你的域名/helloadmin`）
4. 建议删除安装引导文件（`install.php`）以提高安全性


## 使用说明

- **后台管理**：登录后可进行友链管理、审核、站点设置等操作
- **友链申请**：访客可通过`http://yourdomain.com/apply.php`提交友链申请
- **主题设置**：在「网站设置」中可自定义主题颜色、背景图片、玻璃效果等
- **数据备份**：定期通过数据库工具备份数据，确保数据安全


## 常见问题

1. **安装向导提示“fileinfo扩展未启用”**  
   解决：返回步骤1，确保fileinfo扩展已正确安装并重启PHP服务。

2. **数据库连接失败**  
   检查：数据库主机地址是否正确（通常为localhost）、数据库用户名密码是否匹配、数据库是否已创建。

3. **上传图片失败**  
   解决：检查`uploads/`目录是否有可写权限，或图片大小是否超过PHP配置的上传限制。

4. **页面显示错乱**  
   解决：清除浏览器缓存，或检查服务器是否正确加载静态资源（CSS/JS）。
